
## Core Idea

Currently the cubes get rendered one at a time and there is a draw call for each cube
Using a chunk packs cube data and geometry into **one** vbo then **one** draw call per chunk


### 1. Create a Chunk.h

Create the blueprint behind a chunk data type

`CHUNK_SIZE` - How big each chunk will be (cube of CHUNK_SIZE x CHUNK_SIZE)

`chunkX, chunkZ`  - Determines the location of each chunk, Deals in separate coordinates than the main world does (coordinate in main world could be x = 18, z = 20 but in chunk terms you would be in x = 1, z = 1)

`vertices` - This keeps track of all the vertices in the chunk when rendering

`indices` - keeps track of all the indices for those vertices when making the triangles that make up the cubes

`VAO, VBO, EBO` - the variables that each buffer + array lives in

`indexCount` - This stores how many indices the chunk has so render() knows how many to draw. This is needed for the draw call and is basically the size of the vector indices

`generate()` - this generates the chunk and fills out the vectors with vertices + indices. It also uses the noise to generate the y values of each block

`buildMesh()` - This shoves everything into the buffers and effectively builds the mesh that will be drawn

`render()` - This will actually render everything to the screen with a simple draw call using the buffers we set up and the indexCount to tell it how many triangles to render

`cleanup()` - this just frees everything from memory and deletes buffers


```cpp
#pragma once
#include <vector>
#include <glad/glad.h>
#include <glm/glm.hpp>
#include "FastNoiseLite.h"

const int CHUNK_SIZE = 16;

struct Chunk {
    int chunkX, chunkZ;         // chunk's position in chunk-space
    std::vector<float> vertices;
    std::vector<unsigned int> indices;

    unsigned int VAO = 0, VBO = 0, EBO = 0;
    int indexCount = 0;

    void generate(FastNoiseLite& noise);
    void buildMesh();
    void render();
    void cleanup();
};
```



### 2. Create Chunk.cpp

This actually implements each of the functions talked about in the chunk.h section

The extra things here are some cube blueprints that tell specific coordinates for building the cube, and indices that tell how to build each triangle


```cpp
#include "Chunk.h"
#include <cmath>

// Local cube face data — positions only, relative to (0,0,0)
// Each face = 4 vertices (x, y, z)
static const float CUBE_FACES[6][12] = {
    // Front  (z+)
    {-0.5f,-0.5f, 0.5f,  0.5f,-0.5f, 0.5f,  0.5f, 0.5f, 0.5f, -0.5f, 0.5f, 0.5f},
    // Back   (z-)
    { 0.5f,-0.5f,-0.5f, -0.5f,-0.5f,-0.5f, -0.5f, 0.5f,-0.5f,  0.5f, 0.5f,-0.5f},
    // Left   (x-)
    {-0.5f,-0.5f,-0.5f, -0.5f,-0.5f, 0.5f, -0.5f, 0.5f, 0.5f, -0.5f, 0.5f,-0.5f},
    // Right  (x+)
    { 0.5f,-0.5f, 0.5f,  0.5f,-0.5f,-0.5f,  0.5f, 0.5f,-0.5f,  0.5f, 0.5f, 0.5f},
    // Top    (y+)
    {-0.5f, 0.5f, 0.5f,  0.5f, 0.5f, 0.5f,  0.5f, 0.5f,-0.5f, -0.5f, 0.5f,-0.5f},
    // Bottom (y-)
    {-0.5f,-0.5f,-0.5f,  0.5f,-0.5f,-0.5f,  0.5f,-0.5f, 0.5f, -0.5f,-0.5f, 0.5f},
};

// Each face = 2 triangles = 6 indices, offset by the base vertex of that face
static const unsigned int FACE_INDICES[6] = {0, 1, 2, 2, 3, 0};


void Chunk::generate(FastNoiseLite& noise) {
    int worldOffsetX = chunkX * CHUNK_SIZE;
    int worldOffsetZ = chunkZ * CHUNK_SIZE;

    vertices.clear();
    indices.clear();

    unsigned int baseIndex = 0;

    for (int x = 0; x < CHUNK_SIZE; x++) {
        for (int z = 0; z < CHUNK_SIZE; z++) {
            int wx = worldOffsetX + x;
            int wz = worldOffsetZ + z;

            float n = noise.GetNoise((float)wx, (float)wz);
            float y = n * 6.0f;

            // Add all 6 faces for this cube
            for (int f = 0; f < 6; f++) {
                for (int v = 0; v < 4; v++) {
                    vertices.push_back(CUBE_FACES[f][v*3 + 0] + x);
                    vertices.push_back(CUBE_FACES[f][v*3 + 1] + y);
                    vertices.push_back(CUBE_FACES[f][v*3 + 2] + z);
                }
                for (int i = 0; i < 6; i++) {
                    indices.push_back(baseIndex + FACE_INDICES[i]);
                }
                baseIndex += 4;
            }
        }
    }

    buildMesh();
}


void Chunk::buildMesh() {
    if (VAO == 0) {
        glGenVertexArrays(1, &VAO);
        glGenBuffers(1, &VBO);
        glGenBuffers(1, &EBO);
    }

    glBindVertexArray(VAO);

    glBindBuffer(GL_ARRAY_BUFFER, VBO);
    glBufferData(GL_ARRAY_BUFFER,
        vertices.size() * sizeof(float),
        vertices.data(),
        GL_STATIC_DRAW);

    glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, EBO);
    glBufferData(GL_ELEMENT_ARRAY_BUFFER,
        indices.size() * sizeof(unsigned int),
        indices.data(),
        GL_STATIC_DRAW);

    glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE,
        3 * sizeof(float), (void*)0);
    glEnableVertexAttribArray(0);

    glBindVertexArray(0);

    indexCount = (int)indices.size();
}


void Chunk::render() {
    glBindVertexArray(VAO);
    glDrawElements(GL_TRIANGLES, indexCount, GL_UNSIGNED_INT, 0);
}


void Chunk::cleanup() {
    glDeleteVertexArrays(1, &VAO);
    glDeleteBuffers(1, &VBO);
    glDeleteBuffers(1, &EBO);
}
```



### 3. Update the Main C++ File

First include the new chunk header we made
```cpp
#include "Chunk.h"
```

Next we remove these as they are all setup in the chunk instead:
- The vertices array
- The indices array
- cubePositions array
- GRID_WIDTH + GRID_HEIGHT
- Old VAO, VBO, EBO setup + declaration
- getHeight


We then define a world size and generate the chunks before drawing them in the loop
This is made easy with the Chunk.generate method
We do this for each chunk in our chunk coordinate system

```cpp
// 3x3 grid of chunks
const int WORLD_SIZE = 3;
Chunk chunks[WORLD_SIZE][WORLD_SIZE];

for (int cx = 0; cx < WORLD_SIZE; cx++) {
    for (int cz = 0; cz < WORLD_SIZE; cz++) {
        chunks[cx][cz].chunkX = cx;
        chunks[cx][cz].chunkZ = cz;
        chunks[cx][cz].generate(noise);
    }
}
```

We can now update the render loop
we create and pass the model matrix to the shader
Then start rendering each chunk in our chunk coords

```cpp
glUseProgram(shaderProgram);
glUniformMatrix4fv(viewLoc, 1, GL_FALSE, glm::value_ptr(view));
glUniformMatrix4fv(projLoc, 1, GL_FALSE, glm::value_ptr(projection));

// Chunks are pre-built — no per-cube model matrix needed
glm::mat4 model = glm::mat4(1.0f);
glUniformMatrix4fv(modelLoc, 1, GL_FALSE, glm::value_ptr(model));

for (int cx = 0; cx < WORLD_SIZE; cx++) {
    for (int cz = 0; cz < WORLD_SIZE; cz++) {
        chunks[cx][cz].render();
    }
}
```


We can update camera pos to be a bit nicer
```cpp
glm::vec3 cameraPos = glm::vec3(24.0f, 16.0f, 52.0f);
float pitch = -20.0f;
```


Finally we cleanup each chunk before the window terminates

```cpp
for (int cx = 0; cx < WORLD_SIZE; cx++)
    for (int cz = 0; cz < WORLD_SIZE; cz++)
        chunks[cx][cz].cleanup();
```



### Pitfalls to Watch

- Make sure to Remove the old VAO/VBO/EBO
- `glBindVertexArray(0)` at the end of `buildMesh()` is important and forgetting it can cause state leak into subsequent GL calls
- Chunk Positions are in Chunk space chunkx = 1 means offset of x = 16
- All face data is now baked - there is no model matrices per cube, all positions are in the VBO
