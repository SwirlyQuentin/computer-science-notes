In this part, we add per-face normals to the mesh and implement a directional light in the shader the result is terrain looking 3-dimensional + shaded


#### What is a Normal

A normal is a vector that points perpendicular to a surface
- For a flat face, every point on that face has the same normal 

As an example, for the top face of a cube, the normal is the vector that points straight up (0, 1, 0)

This is what `FACE_NORMALS` defines

#### How it Interacts With Light

Light has a direction from which it is coming from
- In this case it is (0.6, 1.0, 0.4)

In the fragment shader, doing a dot product between the normal and the light direction **gives a value between 0 and 1 based on how aligned the two vectors are**

- Normal points directly at light -> dot = 1.0 -> fully lit
- Normal points perpendicular to the light -> dot = 0.0 -> no light
- Normal points away from the light -> dot = negative -> clamped to 0 by max

#### How This Looks on Cubes

The more aligned a cubes face is with the light, the brighter the cube will be

#### What is Ambient Lighting

Ambient lighting is "fake" background light
- It is a constant minimum brightness applied everywhere regardless of normals + light direction
(Simulates light bouncing)



### 1. Define Face Normals

Each face has a fixed normal

```cpp
static const float FACE_NORMALS[6][3] = {
    { 0.0f,  0.0f,  1.0f},  // Front
    { 0.0f,  0.0f, -1.0f},  // Back
    {-1.0f,  0.0f,  0.0f},  // Left
    { 1.0f,  0.0f,  0.0f},  // Right
    { 0.0f,  1.0f,  0.0f},  // Top
    { 0.0f, -1.0f,  0.0f},  // Bottom
};
```


### 2. Pack Normals Into the VBO

Vertex layout goes form 3 floats to 6 floats per vertex (3 for pos + 3 for normals)

Update generate in chunk to put normals in vertices vector

```cpp
for (int f = 0; f < 6; f++) {
    for (int v = 0; v < 4; v++) {
        // Position
        vertices.push_back(CUBE_FACES[f][v*3 + 0] + wx);
        vertices.push_back(CUBE_FACES[f][v*3 + 1] + y);
        vertices.push_back(CUBE_FACES[f][v*3 + 2] + wz);
        // Normal
        vertices.push_back(FACE_NORMALS[f][0]);
        vertices.push_back(FACE_NORMALS[f][1]);
        vertices.push_back(FACE_NORMALS[f][2]);
    }
    for (int i = 0; i < 6; i++) {
        indices.push_back(baseIndex + FACE_INDICES[i]);
    }
    baseIndex += 4;
}
```


### 3. Update Mesh Building Attribute Pointers

The stride is now 6 * the size of a float (bc 3 more floats per vertex)
Also putting in normal information directly into the shader at location 1

```cpp
// Position (location 0)
glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE,
    6 * sizeof(float), (void*)0);
glEnableVertexAttribArray(0);

// Normal (location 1)
glVertexAttribPointer(1, 3, GL_FLOAT, GL_FALSE,
    6 * sizeof(float), (void*)(3 * sizeof(float)));
glEnableVertexAttribArray(1);
```


### 4. Update Vertex Shader

Grab the normals from location 1 and send them to the fragment shader using out

```glsl
#version 330 core
layout (location = 0) in vec3 aPos;
layout (location = 1) in vec3 aNormal;

uniform mat4 view;
uniform mat4 projection;
uniform mat4 model;

out float vHeight;
out vec3 vNormal;
out vec3 vFragPos;

void main(){
    vec4 worldPos = model * vec4(aPos, 1.0);
    gl_Position = projection * view * worldPos;
    vHeight = worldPos.y;
    vNormal = aNormal;
    vFragPos = vec3(worldPos);
}
```


### 5. Update the Fragment Shader

Take in the normals from the vertex shader

Take in uniform light direction + light color from main.cpp

First diffuse lighting
- normalize the normal vectors + light direction
- Then do the dot product to get the value between 0-1 that shows how aligned the vectors are

Second combine the ambient + diffuse
(combining the base amount of light in the scene + the projected amount of light that should be on the face)

Both the ambient light and diffuse light are multiplied by the light color to get the scaled amount of light color

This is then multiplied against the already established color based on the height of the block

```glsl
#version 330 core
in float vHeight;
in vec3 vNormal;
in vec3 vFragPos;

out vec4 FragColor;

uniform vec3 lightDir;      // direction light is coming FROM
uniform vec3 lightColor;

void main(){
    // Height based color
    vec3 color;
    if      (vHeight < 0.0) color = vec3(0.2, 0.4, 0.9);
    else if (vHeight < 2.0) color = vec3(0.3, 0.7, 0.3);
    else if (vHeight < 4.0) color = vec3(0.5, 0.4, 0.3);
    else                    color = vec3(0.9, 0.9, 0.9);

    // Diffuse lighting
    vec3 norm = normalize(vNormal);
    vec3 ld = normalize(lightDir);
    float diff = max(dot(norm, ld), 0.0);

    // Ambient + diffuse combined
    vec3 ambient = 0.2 * lightColor;
    vec3 diffuse = diff * lightColor;

    vec3 result = (ambient + diffuse) * color;
    FragColor = vec4(result, 1.0);
}
```



### 6. Set Light Uniforms From Main

Set the light direction + light color from the main cpp + pass them into the fragment shader

In render loop:
```cpp
// Light coming from upper-right, slightly in front
glm::vec3 lightDir = glm::normalize(glm::vec3(0.6f, 1.0f, 0.4f));
glm::vec3 lightColor = glm::vec3(1.0f, 1.0f, 1.0f);

glUniform3fv(glGetUniformLocation(shaderProgram, "lightDir"),
    1, glm::value_ptr(lightDir));
glUniform3fv(glGetUniformLocation(shaderProgram, "lightColor"),
    1, glm::value_ptr(lightColor));
```



### Pitfalls to Watch

- `glGetUniformLocation` in the render loop is fine for now -- Caching locations is a later optimization
- Normals are not transformed by the matrix model
- Stride 6 must be everywhere that deals with the VBO
- Ambient prevents fully black faces
