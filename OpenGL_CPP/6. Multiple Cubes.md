
### 1. Add a Model Matrix

Right now there is only a view + projection matrix
There is no model matrix that deals with object transform

Update vertex shader to include a new matrix
For the multiplication:
**THE ORDER MATTERS**

```cpp
#version 410 core
layout (location = 0) in vec3 aPos;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main() {
    gl_Position = projection * view * model * vec4(aPos, 1.0);
}
```


### 2. Get Model Uniform Location

Get the location of the model matrix

```cpp
unsigned int modelLoc = glGetUniformLocation(shaderProgram, "model");
```



### 3. Render Multiple Cubes

Keep track of different cube positions where the cube we created earlier will be located

5 cubes, 5 different locations
```cpp
glm::vec3 cubePositions[] = {
    glm::vec3(0.0f,  0.0f,  0.0f),
    glm::vec3(2.0f,  0.0f,  0.0f),
    glm::vec3(-2.0f, 0.0f,  0.0f),
    glm::vec3(0.0f,  2.0f,  0.0f),
    glm::vec3(0.0f, -2.0f,  0.0f)
};
```



### 4. Loop and Draw

Inside the render loop, after binding the vertex array, loop through the cube positions and:
1. Create a model matrix for the cube and set all values to 1
2. use the glm::translate function to translate the model by the position of the cube
	- This will change the model matrix
3. pass the model to the shader
4. draw the model
5. repeat for all other cubes


```cpp
glUseProgram(shaderProgram);
glBindVertexArray(VAO);

for (int i = 0; i < 5; i++) {
    glm::mat4 model = glm::mat4(1.0f);
    model = glm::translate(model, cubePositions[i]);

    glUniformMatrix4fv(modelLoc, 1, GL_FALSE, glm::value_ptr(model));

    glDrawElements(GL_TRIANGLES, 36, GL_UNSIGNED_INT, 0);
}
```
