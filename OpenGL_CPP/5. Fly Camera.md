### 2. Define Camera State

Define
- The position of the camera in space
- The direction the front (horizontal axis) of the camera is facing
- The direction of the vertical axis (up) is facing

```cpp
glm::vec3 cameraPos   = glm::vec3(0.0f, 0.0f, 3.0f);
glm::vec3 cameraFront = glm::vec3(0.0f, 0.0f, -1.0f);
glm::vec3 cameraUp    = glm::vec3(0.0f, 1.0f, 0.0f);
```


### 3. Add Keyboard Movement

Inside the render method, check which key is pressed then move in that specified direction 

For forward and back:
- add the camera speed * the direction of the cameraFront or -cameraFront (depending on which key was pressed) to the camera position

For side to side:
- Cross product of 2 vectors gives a third vector that is perpendicular to both the original vectors
- This results in a vector normal to the plane connecting the original vectors
- Taking the cross product of cameraFront and cameraUp should give a vector that is basically cameraRight
- Taking cross product of up front gives cameraLeft, or you can just do -cameraRight
- The result after normalizing it is added to the camera pos


### 4. Mouse Look

Add globals to deal with the mouse movement
```cpp
float yaw = -90.0f;
float pitch = 0.0f;
float lastX = 400, lastY = 300;
bool firstMouse = true;
```


Then we make a callback function that gets called whenever the mouse is moved in the window

This will 
1. If this is the first time the callback is called, the lastX and lastY are set to the current x and y pos, then cancels firstMouse
2. Find the x and y offset from current position to last position
	- xpos - lastx gives the amount for the yaw to move because its left to right
	- lasty - ypos gives amount for pitch to move because the origin is in the top left making y flipped
3. Multiply these offsets by the sensitivity of the mouse to get the final numbers the pitch and yaw move
4. Add the xoffset to yaw, yoffset to pitch
5. Cap the pitch to 90 degrees both up and down so you cant look past your feet
6. Set the direction of the cameras front to a new direction calculated with the pitch and yaw
	- When on the xz (horizontal plane) length of the projection of the direction is cos(pitch)
		- When looking up, the length of the direction gets smaller
	- Rotate around y-axis using yaw
		- Just circle:
			- cos(yaw) -> x position on circle
			- sin(yaw) -> z position on circle
		- Scaled by cos(pitch) so it shrinks
	- LOOK MORE IN DEPTH
7. Register the callback function and set the cursor to be hidden in the window

```cpp
void mouse_callback(GLFWwindow* window, double xpos, double ypos) {
    if (firstMouse) {
        lastX = xpos;
        lastY = ypos;
        firstMouse = false;
    }

    float xoffset = xpos - lastX;
    float yoffset = lastY - ypos;
    lastX = xpos;
    lastY = ypos;

    float sensitivity = 0.1f;
    xoffset *= sensitivity;
    yoffset *= sensitivity;

    yaw += xoffset;
    pitch += yoffset;

    if (pitch > 89.0f) pitch = 89.0f;
    if (pitch < -89.0f) pitch = -89.0f;

    glm::vec3 direction;
    direction.x = cos(glm::radians(yaw)) * cos(glm::radians(pitch));
    direction.y = sin(glm::radians(pitch));
    direction.z = sin(glm::radians(yaw)) * cos(glm::radians(pitch));

    cameraFront = glm::normalize(direction);
}
```

```cpp
glfwSetCursorPosCallback(window, mouse_callback);
glfwSetInputMode(window, GLFW_CURSOR, GLFW_CURSOR_DISABLED);
```


### 5. Create view + [[Projection Matrix||Projection Matrices]]

The view matrix is the transform that moves the entire world so it looks like the camera is moving

for 
```cpp
glm::mat4 view = glm::lookAt(cameraPos, cameraPos + cameraFront, cameraUp);
```

`cameraPos` - where the camera is

`cameraPos + cameraFront` - where camera is looking so **target = cameraPos + cameraFront**

`cameraUp` - which way is up

`lookAt` computes 3 vectors, forward, right, and up all corrected then builds a matrix that will rotate the world to match orientation + translate world opposite your position

The projection matrix is a transform that converts the 3D space into clip space so that distant objects are smaller and there is perspective

for:
```cpp
glm::mat4 projection = glm::perspective(  
glm::radians(45.0f),  
800.0f / 600.0f,  
0.1f,  
100.0f  
);
```

`radians(45.0f)` - This is the FOV 

`800.0f / 600.0f` - aspect ratio of the screen

`0.1f` - near plane, or the closest distance you can see

`100.0f` - far plane, or the farthest distance you can see


```cpp
glm::mat4 view = glm::lookAt(cameraPos, cameraPos + cameraFront, cameraUp);

glm::mat4 projection = glm::perspective(
    glm::radians(45.0f),
    800.0f / 600.0f,
    0.1f,
    100.0f
);
```



### 6. Update Vertex Shader

Line by line explained:

`layout (location = 0) in vec3 aPos`

`in` - this input data is coming from CPU (the vertex buffer)

`vec3` - each vertex has 3 floats

`aPos` - the variable name inside the shader

`location = 0` - binds this to attribute slot 0

This matches the c++ line: 
```cpp
glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
```

basically saying attribute 0 is position data

```glsl
uniform mat4 view; 
uniform mat4 projection;
```

These are uniform data types -> same value for *every vertex* in this draw call
`mat4` - 4x4 matrix

Represent:
- View - camera transform 
- Projection - perspective transform


```glsl
gl_Position = projection * view * vec4(aPos, 1.0);
```

`vec4(aPos, 1.0);` - this is converting the vector3 to a vector 4 by adding on 1.0 as the value. Vec4 are needed for matrix math (homogeneous coordinates)


The multiplication applies the transformations (Projection transform, view transform)


`gl_Position` - outputs to the gpu, is the built in output variable
- Tells gpu, this is where vertex should appear on screen (in clip space)


### 7. Pass Matrices to Shader

These get the location of the view and projection variables in the shader program and set them to an appointed value

```cpp
unsigned int viewLoc = glGetUniformLocation(shaderProgram, "view");
unsigned int projLoc = glGetUniformLocation(shaderProgram, "projection");

glUniformMatrix4fv(viewLoc, 1, GL_FALSE, glm::value_ptr(view));
glUniformMatrix4fv(projLoc, 1, GL_FALSE, glm::value_ptr(projection));
```
