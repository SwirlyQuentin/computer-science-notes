
### 1. Replace Cube Positions with a Grid

Positions will be calculated based on the width + depth rather than hard coded in
Defined near the top of main

```cpp
// Define grid size at the top of main(), near cameraPos
const int GRID_WIDTH = 16;
const int GRID_DEPTH = 16;
```


### 2. Replace Drawing Section in Render Loop

Loop through both width(x) and depth(z) of grid to place cubes
Each cube is 1 unit
Uses the same way to draw, but is translated according to its coordinates

```cpp
glBindVertexArray(VAO);

for (int x = 0; x < GRID_WIDTH; x++) {
    for (int z = 0; z < GRID_DEPTH; z++) {
        glm::mat4 model = glm::mat4(1.0f);
        model = glm::translate(model, glm::vec3((float)x, 0.0f, (float)z));
        glUniformMatrix4fv(modelLoc, 1, GL_FALSE, glm::value_ptr(model));
        glDrawElements(GL_TRIANGLES, 36, GL_UNSIGNED_INT, 0);
    }
}
```

### 5. Change Control Style

Instead of camera moving where you look when you press w, make it so it just moves forward in relation to world space

Done by taking the cameraFront vector and forgoing the y component + normalizing
The vector is then in just the space of xz

Space and shift are added to go up and down using the cameraUp vector

```cpp
//WASD Input
glm::vec3 flatFront = glm::normalize(glm::vec3(cameraFront.x, 0.0f, cameraFront.z));
if (glfwGetKey(window, GLFW_KEY_W) == GLFW_PRESS){
cameraPos += cameraSpeed * flatFront;
}
if (glfwGetKey(window, GLFW_KEY_S) == GLFW_PRESS){
cameraPos -= cameraSpeed * flatFront;
}
if (glfwGetKey(window, GLFW_KEY_A) == GLFW_PRESS){
cameraPos -= glm::normalize(glm::cross(cameraFront, cameraUp)) * cameraSpeed;
}
if (glfwGetKey(window, GLFW_KEY_D) == GLFW_PRESS){
cameraPos += glm::normalize(glm::cross(cameraFront, cameraUp)) * cameraSpeed;
}
if (glfwGetKey(window, GLFW_KEY_SPACE) == GLFW_PRESS){
cameraPos += cameraSpeed * cameraUp;
}
if (glfwGetKey(window, GLFW_KEY_LEFT_SHIFT) == GLFW_PRESS){
cameraPos -= cameraSpeed * cameraUp;
}
```



### Pitfalls
- Make sure to declare model inside the loop 
- Always construct a fresh mat4 per iteration of placing cube
- Cubes are 1 unit centered at origin
	- Translating by (x, 0, z) places them edge to edge correctly because vertices span [-0.5, 0.5]
