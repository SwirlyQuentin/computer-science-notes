
### 1. Add a getHeight Function

Add a get height function to return a y value when provided an x + z value
This is used to determine the height of the block when placed

cmath is needed for sin + cos functions
```cpp
#include <cmath>
```

```cpp
float getHeight(int x, int z) {
    return std::sin(x * 0.5f) * 2.0f + std::cos(z * 0.5f) * 2.0f;
}
```

The function `sin(x * 0.5) * 2.0f + cos(z * 0.5) * 2.0f` is creating a wave pattern in the blocks

`x * 0.5` this scales down the input of the sin + cos values resulting in controlling the frequency or **how often the wave repeats**

`sin(...) * 2.0f` controls the amplitude of the waves or **how tall/deep the waves are**. Sin oscillates between -1 and 1 so multiplying by 2 allows for oscillation between -2 and 2

`sin() + cos()` - One travels along the x axis and the other travels along the z, this makes it so the height differs and waves appear. If it was just one, it would be a uniform wave across the plane with one height


### 2. Integrate into the Grid Loop

Get the height based on the xz and apply the y value to the translate transformation

```cpp
for (int x = 0; x < GRID_WIDTH; x++) {
    for (int z = 0; z < GRID_DEPTH; z++) {
        float y = getHeight(x, z);

        glm::mat4 model = glm::mat4(1.0f);
        model = glm::translate(model, glm::vec3((float)x, y, (float)z));
        glUniformMatrix4fv(modelLoc, 1, GL_FALSE, glm::value_ptr(model));
        glDrawElements(GL_TRIANGLES, 36, GL_UNSIGNED_INT, 0);
    }
}
```
