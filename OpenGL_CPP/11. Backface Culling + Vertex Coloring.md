
### 1. Enable Backface Culling

This makes it so that the faces the camera doesn't see don't get rendered
It does this by checking which faces are rendered CounterClockwise vs Clockwise
Only CCW faces are front facing

It knows the direction because after it transforms the matrices is when it draws, and anything front facing is CCW

After `glEnable(GL_DEPTH_TEST)`:

```cpp
glEnable(GL_CULL_FACE);
glCullFace(GL_BACK);
glFrontFace(GL_CCW);
```


### 2. Coloring Terrain According to Height

This can be done in the cpp files, but I did it in the shader files

First in the vertex shader, pass the height of the current drawing position to the fragment shader

Make sure to adjust the vHeight in according to the world height, this just means multiplying by the model matrix in order to get correct world transformations

```glsl
out float vHeight;  // add this

void main(){
    // your existing gl_Position line stays
    vec4 worldPos = model * vec4(aPos, 1.0);
    vHeight = worldPos.y;  // add this
}
```

Then in the fragment shader, use that height to color the cubes accordingly

You recieve the height and color based on the height
lower = water, higher = varying terrain types
This is temporary just for some visual clarity

```glsl
#version 330 core
out vec4 FragColor;
in float vHeight;  // receive from vertex shader

void main(){
    vec4 water  = vec4(0.2, 0.4, 0.9, 1.0);  // blue
    vec4 land   = vec4(0.3, 0.7, 0.3, 1.0);  // green
    vec4 hills  = vec4(0.5, 0.4, 0.3, 1.0);  // brown
    vec4 peak   = vec4(0.9, 0.9, 0.9, 1.0);  // white

    if      (vHeight < 0.0) FragColor = water;
    else if (vHeight < 2.0) FragColor = land;
    else if (vHeight < 4.0) FragColor = hills;
    else                    FragColor = peak;
}
```
