### The Process Concept

- A [[Process]] is an instance of a program being executed by an OS.
- The OS itself is organized as a collection of processes.
- A program can be executed multiple times simultaneously:
  - Each execution instance is a separate process.
  - Each process has its own [[PCB]].
  - Each [[PCB]] contains a separate [[Program Counter]] to track execution location.

---

#### [[PCB]] (Process Control Block)

- The OS keeps track of each process using a [[PCB]].
- The [[PCB]] is an independent data structure managed by the OS.
- It is the concrete representation of a process.
- The OS creates a new PCB for every new process.

##### The PCB includes:

- The current instruction address (via [[Program Counter]])
- The execution stack
- The set of resources used by the process
- The program being executed
- The current state of the process

##### Possible Process States (Basic Model):

- Running
- Ready
- Blocked

---

### Process States and Transitions

#### Basic Process Lifecycle

- Newly created processes start in the Ready state.
- OS selects process for execution → state changes to Running.
- OS interrupts process (to let another process use the CPU) → state changes to Ready.
- OS restarts process → state changes to Running.
- Process requests an unavailable resource (e.g., file already open) → state changes to Blocked.
- Resource is released → state changes to Ready.

---

#### Advanced Process States and Transitions

Some operating systems provide a more complex set of states and transitions.

##### New State

- A newly created process may first enter the New state.
- The OS may regulate how many processes compete for the CPU.

##### Terminated State

- A process enters the Terminated state when execution can no longer continue.
- The process remains here temporarily before its [[PCB]] is deleted.
- The OS may inspect the final state (e.g., after a fatal error).

##### Suspended State

- A process may be placed into the Suspended state even if CPU and resources are available.
- Used for:
  - Debugging
  - Performance regulation
  - Administrative control

---

### The Context Switch

- The CPU executes ONE process at a time.
- Multiple processes share a CPU by taking turns (time-sharing).

#### [[Context Switch]]

- A [[Context Switch]] is the transfer of control from one process to another.
- It is triggered by the OS (e.g., timer interrupt, system call, blocking event).

##### What Happens During a Context Switch?

When a process stops executing:

1. The OS saves all information about the process.
2. This information is stored in its [[PCB]].
3. When the process runs again, the OS restores this information.

---

#### [[CPU State]]

The information that must be saved during a context switch is called the [[CPU State]].

It consists of:

- All intermediate values in every [[CPU Register]]
- Hardware flags
- The [[Program Counter]] (which holds the address of the next instruction)
