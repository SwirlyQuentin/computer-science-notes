# **Batch Processes**

A **batch process** performs a long-running and generally repetitive task that does not require any intervention from the user.

## **Examples**

* Payroll
* Insurance claims processing
* Weather prediction
* Scientific calculations

---

# **[[FIFO Scheduling Algorithm]] (FCFS)**

The **FIFO (First-In-First-Out)** or **[[FIFO Scheduling Algorithm|FCFS]] (First-Come-First-Serve)** algorithm schedules processes strictly according to [[Process]] arrival time.

## **Core Idea**

* Earlier arrival ⇒ Higher [[Process Priority|Priority]]

## **Tie-Breaking**

* If multiple processes have the same arrival time:
  * The [[Arbitration Rule]] picks a [[Process]] at random
  * In reality, all requests are processed sequentially and have different arrival times

## **Type**

* [[Non-Preemptive Scheduling|Non-preemptive]]

---

# **[[SJF Scheduling Algorithm]] (SJN)**

The **[[SJF Scheduling Algorithm|SJF]] (Shortest Job First)** or **SJN (Shortest Job Next)** algorithm schedules processes according to total CPU time requirements.

## **Core Idea**

* Shorter required CPU time ⇒ Higher [[Process Priority|Priority]]

## **Tie-Breaking**

* If multiple processes have the same CPU time requirement:
  * The [[Arbitration Rule]] picks one based on arrival time

## **Type**

* [[Non-Preemptive Scheduling|Non-preemptive]]

---

# **[[SRT Scheduling Algorithm]]**

The **[[SRT Scheduling Algorithm|SRT]] (Shortest Remaining Time)** algorithm schedules processes according to the remaining CPU time needed to complete work.

## **Core Idea**

* Shorter remaining CPU time ⇒ Higher [[Process Priority|Priority]]

## **Tie-Breaking**

* If multiple processes have the same remaining CPU time:
  * The [[Arbitration Rule]] picks one based on arrival time

## **Type**

* [[Preemptive Scheduling|Preemptive]]
* This is the preemptive version of [[SJF Scheduling Algorithm]]

---

# **Estimating Total CPU Time**

## **For [[Long-Term Scheduling]]**

* Total CPU time is measured between [[Process Creation]] and destruction
* Estimation methods:
  * Based on previous runs of the same program
  * Provided by the programmer

## **For [[Short-Term Scheduling]]**

* Total [[CPU Burst]] changes for each arrival/departure period
* Total CPU time:
  * Not known in advance
  * Estimated based on past behavior

### **Exponential Averaging**

Combines the previous estimate ( S_n ) and the last observed value ( T_n ):
```
S_{n+1} = α * T_n + (1 - α) * S_n
```

* `α` controls the weight between recent observation ( T_n ) and historical estimate ( S_n )

---

# **Predicting Total CPU Time**

## **When α = 0.8**

* Prediction relies mostly on the last observed value ( T_n )

**Advantages**
* Adapts quickly to changes

**Disadvantages**
* Sensitive to outliers
* Can lead to erratic predictions

## **When α = 0.2**

* Prediction relies mostly on past estimates ( S_n )

**Advantages**
* Smooth predictions
* Ignores outliers

**Disadvantages**
* Slow to adapt to changes

# **Performance of the Algorithms**

## **Primary Goal (Batch Processes)**

* Maximize the number of processes completed per unit of time

## **Key Insight**

* Scheduling cannot reduce total CPU time of individual processes
* It **can reduce waiting time**

---

## **Turnaround Time**

The [[turnaround time]] of a [[Process]] is the time between arrival and departure.

* It includes:
  * Total CPU time
  * Waiting time

### **Average Turnaround Time (ATT)**

* The mean [[turnaround time]] across all processes

---

## **Fairness and [[Starvation]]**

**Fairness**
* Ensuring all processes get a chance to execute

**[[Starvation]]**
* The indefinite postponement of a [[Process]] while other processes are allowed to proceed

**Important Note**
* Both [[SJF Scheduling Algorithm]] and [[SRT Scheduling Algorithm]] can lead to [[starvation]]
