## What is Clipping?

**Clipping** is the process of **removing parts of graphical primitives that fall outside a defined viewing region**.

In rendering, we define a **clipping window (2D)** or **clipping volume (3D)**. Any part of a line, [[polygon]], or object outside this region is discarded, while the portion inside the region is kept.

### Why Clipping is Important
- Prevents drawing geometry that cannot appear on the screen
- Improves rendering efficiency
- Ensures objects only appear inside the viewing window

Example pipeline stage:

```
Scene Geometry → Clipping → Rasterization → Framebuffer → Screen
```

---

# Line Clipping in 2D

## Determining if a Point is Inside the Clip Rectangle

A point is inside the clipping rectangle if

$$
x_{min} < x < x_{max} \quad \text{and} \quad y_{min} < y < y_{max}
$$

---

## Endpoint Analysis for Lines

When examining a line segment, check its two endpoints.

### Cases

**Trivial Acceptance**
- Both endpoints lie inside the clipping region
- Entire line is visible

**Must Clip**
- One endpoint inside
- One endpoint outside

**Uncertain Case**
- Both endpoints outside
- The line **may or may not intersect** the clipping region

---

## Brute Force Line Clipping

A naive approach computes intersections between the line and the **four clip edges**.

Line equation:

$$
y = mx + b
$$

### Problems

- Represents **infinite lines**, not segments
- Does **not handle vertical lines** (undefined slope)
- Requires checking every edge

Because of this, more efficient algorithms are used.

---

# Cohen–Sutherland Line Clipping

The **Cohen–Sutherland algorithm** efficiently clips lines against a rectangular clipping window.

## Key Idea

Divide the plane into **9 regions** around the clip rectangle.

Each region is represented using a **4-bit outcode** describing where a point lies relative to the rectangle.

---

## Computing Outcodes

Each point is tested against the clipping bounds:

1. $y_{max} - y$
2. $y - y_{min}$
3. $x_{max} - x$
4. $x - x_{min}$

A point lies **inside** if all tests are positive.

### Bit Meaning

| Bit | Meaning |
|----|----|
| 1 | Above top edge |
| 2 | Below bottom edge |
| 3 | Right of right edge |
| 4 | Left of left edge |

Each endpoint receives an **outcode**

- $OC_0$ → first endpoint
- $OC_1$ → second endpoint

---

## Trivial Cases

### Trivial Acceptance

If

$$
OC_0 = 0000 \quad \text{and} \quad OC_1 = 0000
$$

Both endpoints are inside → **accept line**

---

### Trivial Rejection

If

$$
(OC_0 \; \& \; OC_1) \ne 0
$$

The endpoints share an outside region → **reject line**

---

# Cohen–Sutherland Clipping Strategy

If the line is **neither trivially accepted nor rejected**, the algorithm clips the line against the window edges.

### Steps

1. Choose the endpoint outside the clipping window.
2. Use its outcode to determine which edge is crossed.
3. Compute the **intersection point** with that edge.
4. Replace the outside endpoint with the intersection point.
5. Recompute the outcode.
6. Repeat until the line is accepted or rejected.

---

## Line Intersection Equations

Useful line forms:

$$
y = y_0 + slope(x - x_0)
$$

$$
x = x_0 + \frac{1}{slope}(y - y_0)
$$

These allow solving intersections when **x or y is fixed by the clipping boundary**.

---

# Cohen–Sutherland Algorithm (Pseudocode)

```
ComputeOutCode(x0, y0, outcode0)
ComputeOutCode(x1, y1, outcode1)

repeat

    check for trivial accept or trivial reject

    pick the endpoint that is outside the clip rectangle

    if TOP then
        x = x0 + (x1 - x0) * (ymax - y0) / (y1 - y0)
        y = ymax

    else if BOTTOM then
        x = x0 + (x1 - x0) * (ymin - y0) / (y1 - y0)
        y = ymin

    else if RIGHT then
        y = y0 + (y1 - y0) * (xmax - x0) / (x1 - x0)
        x = xmax

    else if LEFT then
        y = y0 + (y1 - y0) * (xmin - x0) / (x1 - x0)
        x = xmin

    if (x0, y0 is the outer point)
        x0 = x
        y0 = y
        ComputeOutCode(x0, y0, outcode0)

    else
        x1 = x
        y1 = y
        ComputeOutCode(x1, y1, outcode1)

until line accepted or rejected
```

# Clipping in 3D

The same idea extends to **3D clipping volumes**.

Instead of 9 regions, space is divided into **27 regions**.

Each point receives a **6-bit outcode**.

| Bit | Meaning |
|----|----|
| 1 | Behind back plane |
| 2 | In front of front plane |
| 3 | Above top plane |
| 4 | Below bottom plane |
| 5 | Right of right plane |
| 6 | Left of left plane |

### Acceptance / Rejection Rules

**Trivial Accept**

$$
OC_0 = 0 \quad \text{and} \quad OC_1 = 0
$$

**Trivial Reject**

$$
(OC_0 \; \& \; OC_1) \ne 0
$$

Otherwise the line must be clipped against the relevant planes until it is accepted or rejected.

# Example: Cohen–Sutherland Line Clipping

Given a window size of **1600 × 1000**, with the **origin at the center of the window**, and the following endpoints of a line:

- $P_0 = (-900, 200)$  
- $P_1 = (300, -100)$  

Determine whether the line can be **trivially accepted**, **trivially rejected**, or **must be clipped** using the **Cohen–Sutherland line clipping algorithm**.

### Clipping Window Bounds

Since the origin is centered:

- $x_{min} = -800$
- $x_{max} = 800$
- $y_{min} = -500$
- $y_{max} = 500$

---

# Step 1 — Compute Outcodes

Outcodes are determined by testing each point against the clipping boundaries.

Bit meanings:

| Bit | Meaning |
|----|----|
| 1 | Above top |
| 2 | Below bottom |
| 3 | Right of right edge |
| 4 | Left of left edge |

---

### Outcode for $P_0 = (-900,200)$

Check bounds:

- $x < x_{min}$ → $-900 < -800$ → **true**
- $x > x_{max}$ → false
- $y < y_{min}$ → false
- $y > y_{max}$ → false

So the point lies **to the left of the window**.

Outcode of $P_0$:

```
0001
```

---

### Outcode for $P_1 = (300,-100)$

Check bounds:

- $x < x_{min}$ → false
- $x > x_{max}$ → false
- $y < y_{min}$ → false
- $y > y_{max}$ → false

This point is **inside the window**.

Outcode of $P_1$:

```
0000
```

---

# Step 2 — Determine Acceptance or Rejection

Trivial accept condition:

$$
OC_0 = 0000 \quad \text{and} \quad OC_1 = 0000
$$

Not satisfied.

Trivial reject condition:

$$
(OC_0 \; \& \; OC_1) \neq 0
$$

```
0001 AND 0000 = 0000
```

Not rejected.

Therefore:

**The line must be clipped.**

---

# Step 3 — Determine Which Edge to Clip Against

Since $P_0$ has outcode **0001**, the line crosses the **left clipping boundary**.

Left boundary:

$$
x = x_{min} = -800
$$

---

# Step 4 — Compute Line Equation

Slope:

$$
m = \frac{y_0 - y_1}{x_0 - x_1}
$$

Substitute values:

$$
m = \frac{200 - (-100)}{-900 - 300}
$$

$$
m = \frac{300}{-1200}
$$

$$
m = -0.25
$$

Using slope-intercept form:

$$
y = mx + b
$$

Substitute $P_0$:

$$
200 = (-0.25)(-900) + b
$$

$$
200 = 225 + b
$$

$$
b = -25
$$

Line equation:

$$
y = -0.25x - 25
$$

---

# Step 5 — Find Intersection with Clipping Boundary

Left boundary:

$$
x = -800
$$

Substitute into the line equation:

$$
y = -0.25(-800) - 25
$$

$$
y = 200 - 25
$$

$$
y = 175
$$

---

# Step 6 — Final Clipping Point

The intersection point is:

$$
(-800,\;175)
$$

So the new clipped line segment becomes:

- $(-800,175)$
- $(300,-100)$

---

# Final Result

- Outcode of $P_0$: **0001**
- Outcode of $P_1$: **0000**

The line **cannot be trivially accepted or rejected**.

The line **must be clipped at the left boundary**.

Final clipping point:

$$
(-800,175)
$$
