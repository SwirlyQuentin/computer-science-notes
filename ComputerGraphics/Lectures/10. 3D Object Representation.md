## Building 3D Primitives

- 3D primitives are created from **2D and 1D primitives**.
- **Triangles** are the most commonly used primitive in 3D graphics.
- When many triangles are used to represent a single object, it is called a **triangular mesh**.

Example expression:

$x(s,t)=(1-s)^3(1-t)^3P_{1,1}$

### Splines and Patches

- **Splines** describe the boundaries of surface **patches**.
- These patches can be **sewn together** to represent curved surfaces.

### Polygon Count

- Meshes with **higher polygon counts** behave like a more flexible sheet.
- This results in:
  - **More realistic rendering**
  - **Higher computational cost**
  - **Longer render times**

---

## Triangle Meshes

- The **most common representation** of shape in three-dimensional graphics.
- The **three vertices of a triangle always lie on a single plane**.
- Their **uniform structure** makes them easy to manipulate.

Common mesh operations include:

- Subdivision
- Simplification
- Transformation

There are **multiple ways** to represent triangular meshes.

---

## Triangular Mesh Representation

### Vertex and Face Tables

A common representation uses:

- A **vertex table**
- A **face table**

**Vertex Table**
- Each vertex is stored **once**.

**Face Table**
- Triangles are stored as **ordered triplets of indices** pointing to vertices.

Example:
```
Triangle → (v1, v5, v3)
```

Additional notes:

- **Edges are inferred** from triangle connections.
- Faces are often stored alongside their associated vertices.

### Vertex Ordering

- Vertices in the face table are typically stored in **counter-clockwise order**.
- Consistent ordering helps determine:
  - surface normals
  - front vs back faces
  - correct rendering behavior

---

## Creating 3D Shapes from 2D Shapes

3D shapes are often constructed by **extending 2D shapes into another dimension**.

Examples:

- Square → **Cube**
- Triangle → **Pyramid**

This is done by adding additional surfaces such as:

- left
- right
- back
- bottom

### Object Positioning

For objects to rotate correctly around an axis:

- They should be **modeled around the origin (0,0,0)**.
- Afterward they can be **translated** to their final position.

---

## Drawing a Cube

Drawing a cube is similar to **drawing a square in 3D space** (see [[9. 3D Viewing]]).

### Cube Structure

- A cube is composed of **6 quads** (faces).
- Each quad uses **4 vertices**.

### Vertex Order

It is good practice to maintain **consistent vertex ordering**:

- Either **clockwise**
- Or **counter-clockwise**

In these notes, **counter-clockwise ordering** is used.

For each quad:

1. Top Right
2. Top Left
3. Bottom Left
4. Bottom Right

Even when drawing the **back face**, the order should remain **counter-clockwise relative to the visible surface**.

---

## Using `GL_QUADS`

OpenGL allows drawing quads using:
```java
glBegin(GL_QUADS)
```

Rules:

- Every **4 vertices** form **one quad**.
- Supplying **more or fewer vertices** results in an error.
- After one quad is drawn, the **next 4 vertices create another quad**.

---

## Constructing the Cube

### Cube Position

- The **center of the cube should be at the origin (0,0,0)**.
- The cube extends **1 unit in each direction**.

Example:

- Top face → `y = 1`
- Bottom face → `y = -1`

### Drawing the Top Face

Steps:

1. Move **1 unit up** from the center.
2. Draw a quad along the **Z plane**.
3. The quad extends **toward and away from the viewer**.

Example process:

- Start with the **top-right corner**
- Move **one unit right**
- Move **one unit into the screen**
- Continue defining the remaining vertices

### Drawing the Bottom Face

- The bottom face is identical to the top face but located at:
```
y = -1
```

### Why Vertex Order Matters

If vertex order is ignored:

- The **inside of a face may be rendered as the outside**.
- This becomes especially problematic with:
  - **Texture mapping**
  - **Lighting calculations**
  - **Back-face culling**

---

## Cube Rendering Example
```java
private void render(){
    try{
        glBegin(GL_QUADS);

        // Top
        glColor3f(0.0f, 0.0f, 1.0f);

        glVertex3f(1.0f, 1.0f, -1.0f);
        glVertex3f(-1.0f, 1.0f, -1.0f);
        glVertex3f(-1.0f, 1.0f, 1.0f);
        glVertex3f(1.0f, 1.0f, 1.0f);

        // Bottom
        glVertex3f(1.0f, -1.0f, 1.0f);
        glVertex3f(-1.0f, -1.0f, 1.0f);
        glVertex3f(-1.0f, -1.0f, -1.0f);
        glVertex3f(1.0f, 1.0f, -1.0f);

        // Front
        glVertex3f(1.0f, 1.0f, 1.0f);
        glVertex3f(-1.0f, 1.0f, 1.0f);
        glVertex3f(-1.0f, -1.0f, 1.0f);
        glVertex3f(1.0f, -1.0f, 1.0f);

        // Back
        glVertex3f(1.0f, -1.0f, -1.0f);
        glVertex3f(-1.0f, -1.0f, -1.0f);
        glVertex3f(-1.0f, 1.0f, -1.0f);
        glVertex3f(1.0f, 1.0f, -1.0f);

        // Left
        glVertex3f(-1.0f, 1.0f, 1.0f);
        glVertex3f(-1.0f, 1.0f, -1.0f);
        glVertex3f(-1.0f, -1.0f, -1.0f);
        glVertex3f(-1.0f, -1.0f, 1.0f);

        // Right
        glVertex3f(1.0f, 1.0f, -1.0f);
        glVertex3f(1.0f, 1.0f, 1.0f);
        glVertex3f(1.0f, -1.0f, 1.0f);
        glVertex3f(1.0f, -1.0f, -1.0f);

        glEnd();
    }
    catch(Exception e){
    }
}
```
