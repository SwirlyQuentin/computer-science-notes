# Building 3D Primitives

3D objects in graphics are typically constructed from simpler geometric primitives.

- **3D primitives** are built from **1D and 2D primitives**.
- The most common primitive used is the **triangle**.
- A collection of many triangles used to represent one object is called a **triangular mesh**.

Example expression from notes:

$$
x(s,t) = (1-s)^3 x (1-t)^3 x P_{1,1}
$$

### Surface Construction

- **Splines** are used to describe the boundaries of surface patches.
- These patches can be **sewn together** to represent curved surfaces.

### Polygon Count

- A **higher polygon count** results in:
  - More flexible surface geometry
  - More realistic rendering
- However, more polygons also increase **rendering time and computational cost**.

---

# Triangle Meshes

Triangle meshes are the **most common representation of 3D geometry**.

### Key Properties

- Every triangle has **three vertices**.
- The vertices of a triangle **always lie on a single plane**.
- This consistent structure makes many operations easier:
  - subdivision
  - simplification
  - transformations
  - rendering

Because of this simplicity and consistency, triangles are widely used in **graphics pipelines and GPUs**.

---

# Triangular Mesh Representation

A common mesh representation uses two tables:

### 1. Vertex Table
- Stores all vertices in the mesh.
- Each vertex is stored **once**.

Example vertex:

```
v0 = (x, y, z)
```

### 2. Face Table
- Stores triangles using **indices into the vertex table**.
- Each triangle is stored as an **ordered triplet** of vertex indices.

Example:

```
Triangle = (v1, v5, v8)
```

### Important Notes

- **Edges are inferred** from the triangle index data.
- Face associations are often stored with vertices.
- Vertices in the face table are listed in **counter-clockwise (CCW) order**.

The winding order helps determine:

- surface normals
- front vs back faces
- correct lighting and culling

# Building 3D Shapes from 2D Shapes

2D shapes can be extended into 3D objects by adding additional surfaces.

Examples:

| 2D Shape | 3D Shape |
|--------|--------|
| Square | Cube |
| Triangle | Pyramid |

### General Process

1. Start with the base 2D shape.
2. Add additional faces (left, right, back, etc.).
3. Connect edges to form a closed 3D object.

### Object Positioning

If an object needs to **rotate around an axis**:

- Design it **centered around the origin**:

```
(0,0,0)
```

- Then **translate it to the desired position**.

This simplifies rotation transformations.

# Drawing a Cube

A cube is constructed from **6 quadrilateral faces (quads)**.

This is similar to drawing a square in 3D space.

Important rule:

- Maintain a **consistent vertex winding order** (clockwise or counter-clockwise).

---

# Using Quads in OpenGL

Quads can be drawn using:

```
glBegin(GL_QUADS)
```

### Rules for Quads

- Every **4 vertices define one quad**.
- Providing **fewer or more than 4 vertices** will cause rendering issues.
- After one quad is defined, the next **4 vertices define the next quad**.

---

# Winding Order and Face Orientation

In the lecture example, vertices are defined in **counter-clockwise order**.

Typical vertex order for a quad:

1. top-right
2. top-left
3. bottom-left
4. bottom-right

### Important Note

The **back face** of the cube may appear clockwise from some viewpoints, but it is still **counter-clockwise when viewed from the outside of the cube**.

Correct winding order is important for:

- face culling
- lighting calculations
- normal direction

---

# Constructing the Cube

### Step 1 — Start With the Top Face

- The cube is centered at the origin.
- Move **1 unit upward** to form the top face.

```
y = 1
```

- Draw the quad across the **z-axis** (into and out of the screen).

### Step 2 — Build Remaining Faces

The rest of the cube is built by defining faces that connect the corresponding vertices.

### Bottom Face

- Same shape as the top face
- Positioned **1 unit downward**

```
y = -1
```

### Winding Order Considerations

If the winding order is ignored:

- Coordinates may be reused more easily
- But this can cause **texture mapping problems**

For example:

- The **outside of the quad** may be interpreted as the **inside**.

# Example Code (Lecture Version)

```java
private void render(){
	try{
		glBegin(GL_QUADS);

			// Top
			glColor3f(0.0f, 0.0f, 1.0f);
			glVertex3f(1.0f, 1.0f, -1.0f);
			glVertex3f(-1.0f, 1.0f, -1.0f);
			glVertex3f(-1.0f, 1.0f, 1.0f);
			glVertex3f(1.0f, 1.0f, 1.0f);

			// Bottom
			glVertex3f(1.0f, -1.0f, 1.0f);
			glVertex3f(-1.0f, -1.0f, 1.0f);
			glVertex3f(-1.0f, -1.0f, -1.0f);
			glVertex3f(1.0f, 1.0f, -1.0f);

			// Front
			glVertex3f(1.0f, 1.0f, 1.0f);
			glVertex3f(-1.0f, 1.0f, 1.0f);
			glVertex3f(-1.0f, -1.0f, 1.0f);
			glVertex3f(1.0f, -1.0f, 1.0f);

			// Back
			glVertex3f(1.0f, -1.0f, -1.0f);
			glVertex3f(-1.0f, -1.0f, -1.0f);
			glVertex3f(-1.0f, 1.0f, -1.0f);
			glVertex3f(1.0f, 1.0f, -1.0f);

			// Left
			glVertex3f(-1.0f, 1.0f, 1.0f);
			glVertex3f(-1.0f, 1.0f, -1.0f);
			glVertex3f(-1.0f, -1.0f, -1.0f);
			glVertex3f(-1.0f, -1.0f, 1.0f);

			// Right
			glVertex3f(1.0f, 1.0f, -1.0f);
			glVertex3f(1.0f, 1.0f, 1.0f);
			glVertex3f(1.0f, -1.0f, 1.0f);
			glVertex3f(1.0f, -1.0f, -1.0f);

		glEnd();
	}
	catch(Exception e){
	}
}
```

# Key Takeaways

- 3D objects are typically represented using **triangle meshes**.
- Triangles are used because they **always lie on a plane**.
- Meshes are often stored using **vertex tables and face tables**.
- Consistent **winding order** is critical for correct rendering.
- Complex 3D shapes are built from **simpler primitives** such as quads and triangles.
